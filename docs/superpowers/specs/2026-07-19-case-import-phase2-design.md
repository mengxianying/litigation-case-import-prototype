# 案件导入二期设计说明

## 1. 目标与边界

二期在一期 Excel 导入能力之上补齐材料包上传、材料解析、规则配置和导入处理闭环，不改变一期已经生成并生效案件的可用状态。

二期上线后，案件导入管理与新建导入批次恢复为最终目标态；导入处理详情、导入规则配置和新建导入规则作为独立菜单承接处理与配置。

本次不引入资产方字段别名。各资产方仍使用统一的中文列头 Excel 模板；字段必填、材料类别和材料必填规则由导入规则配置维护。

## 2. 核心口径

### 2.1 案件激活状态

字段校验通过后，系统创建案件记录，并根据材料校验结果确定案件是否生效。

| 场景 | 案件状态 | 是否进入待委外 | 可操作项 |
| --- | --- | --- | --- |
| 字段校验失败、主续关系失败、金额异常、重复订单不可覆盖 | 导入失败 | 否 | 下载明细；修正 Excel 后重新导入 |
| 字段校验通过，必填材料齐全 | 已生效 | 是 | 后续法诉流程；可补充非必填材料 |
| 字段校验通过，缺少任一必填材料 | 待材料激活 | 否 | 在导入处理详情批量补传必填材料；补齐后自动生效 |
| 字段校验通过，必填材料齐全，但缺非必填材料 | 已生效 + 材料待补 | 是 | 在导入处理详情批量补传非必填材料 |

“待材料激活”是二期新增的案件状态，不等同于“导入失败”。它表达案件基础数据已经可落库，但尚未具备进入待委外的证据材料条件。

### 2.2 一期兼容

1. 一期已完成导入且已生效的案件，二期上线后继续保持“已生效”，不因未上传材料而回溯为待材料激活。
2. 一期历史批次的材料状态统一展示为“未校验（一期导入）”；不生成待补任务，也不影响其待委外与后续法诉流程。
3. 一期批次的字段校验、重复订单、主续订单及金额校验口径保持不变。二期仅在新上传材料包的批次上增加材料解析与激活判断。
4. 一期导入案件继续不支持导出要素表、导出法诉材料；二期新增材料包且材料已解析的案件，可按后续法诉材料能力评估开放导出。

## 3. 页面与交互调整

### 3.1 案件导入管理

恢复材料包、导入处理详情、草稿、材料状态、错误明细等目标态能力。列表新增或统一以下口径：

| 字段/动作 | 二期口径 |
| --- | --- |
| 批次状态 | 草稿、处理中、导入完成、导入失败。批次中存在成功或待材料激活订单即为“导入完成”；全部订单均未处理成功时为“导入失败”。 |
| 案件生效情况 | 已生效、待材料激活、导入失败。展示订单数量汇总，可进入导入处理详情查看订单明细。 |
| 材料状态 | 未校验（一期）、材料齐全、待补必填材料、材料待补、存在未匹配材料。 |
| 编辑 | 可修改批次基础信息、补传资产明细或材料包。历史文件只追加，不覆盖原始上传记录。 |
| 导入处理详情 | 对处理中、导入完成和导入失败批次可进入；按订单问题、待补传材料、未匹配材料统一处理。 |

编辑批次再次导入 Excel 时，仍按订单编号判断重复：未生效或已生效但仍在待分配队列的订单可使用新 Excel 覆盖字段；已分配或已进入后续法诉节点的订单不能覆盖，写入结果明细并提示订单当前所在状态。

### 3.2 新建导入批次

新增导入规则选择和材料包上传。保存草稿仅保存批次基础信息与临时文件，不解析、不生成案件。提交导入后系统固化规则快照，后续规则修改不影响该批次的字段和材料校验结果。

材料包上传支持压缩包；系统按订单编号、资产方材料名称、匹配模式（精确/包含）和标准材料类别解析。解析失败、无法识别订单或无法匹配类别的文件进入未匹配材料池，不阻断其他文件的解析。

### 3.3 导入处理详情

将字段和材料问题收敛在一个页面，使用三个页签：

| 页签 | 内容 | 可操作项 |
| --- | --- | --- |
| 订单处理 | 字段失败、主续关系失败、重复订单不可覆盖、落库异常的订单 | 查看解析、下载明细、重新导入失败订单 |
| 待补传材料 | 待材料激活订单缺失的必填材料，以及已生效订单缺失的非必填材料 | 批量补传；补必填材料后触发材料复核与自动生效 |
| 未匹配材料 | 已上传但无法归属订单或无法归类的文件 | 查看/下载文件、重新识别；二期不提供人工绑定订单或单文件上传 |

“批量补传”根据当前页签自动限定可上传范围：

1. 在待材料激活订单下，可补传当前规则配置的缺失必填材料和非必填材料。
2. 在已生效订单下，仅可补传非必填材料，不改变案件生效状态。
3. 失败订单的字段问题不通过材料补传处理，需由“重新导入失败订单”携带原批次基础信息、规则快照后重新上传 Excel。

### 3.4 导入规则配置与新建规则

规则按“外部资产方 + 业务类型（租赁）”维护。新增资产方时以统一 70 列模板和标准材料类别生成初始规则，运营只编辑启用/必填及资产方材料名称，不可删除系统锁定基础字段和标准材料类别。

Excel 模板字段配置展示“模板字段 / 对应表名与字段名 / 是否启用及必填”三类信息。订单编号、姓名、身份证号、债权金额、违约日为锁定启用且必填；商品名称、下单手机号、业务主体1、主合同签署日期、租赁物签收日为默认必填，可按规则调整。

首租与续租只在材料规则配置中切换。两类订单共用同一套 Excel 字段配置，但可配置不同的必填材料清单。规则在批次提交时生成快照，后续修改仅对新批次生效。

## 4. 主续订单与重复订单

1. 首次导入续租订单时，关联主订单必须同批次导入或已存在于系统，且客户证件号一致；主订单以已结清历史订单导入，不生成债权金额、已还金额、剩余未还、违约日、逾期天数等法诉债权字段。
2. 若主订单已在法诉系统形成案件，后续导入其续租订单时阻断，订单明细提示：`续租订单关联的主订单【{父订单编号}】已在法诉系统形成案件，不符合续租订单导入条件。`
3. 同一 Excel 内订单编号重复时，上传阶段阻断整份文件，不创建导入任务，提示“模板中有重复订单，请修正后重新上传。”
4. 同一批次多次补传 Excel 时，订单数量以订单编号去重统计；不可覆盖订单写入结果明细并展示已存在的批次和案件状态。

## 5. 备注呈现要求

二期相关页面右侧备注以 `【二期调整】` 标识新增或变化内容，至少写清：

1. 页面筛选字段和重点列表字段的取数逻辑。
2. 案件生效、待材料激活、材料待补、未匹配材料和导入失败的判定口径。
3. 每个按钮的可操作状态、处理结果与禁止条件。
4. 与一期历史批次的兼容规则，避免运营误将“未校验（一期导入）”理解为缺失材料。
5. 规则快照、重新导入、材料补传和自动生效的边界。

## 6. 验收标准

1. 缺必填材料的订单可在处理详情补齐，补齐且复核通过后自动变为已生效并进入待委外。
2. 缺非必填材料的订单保持已生效，仅展示材料待补，不影响后续流程。
3. 一期历史已生效案件不被二期材料规则影响，材料状态展示“未校验（一期导入）”。
4. 规则修改后，历史批次仍按提交时的规则快照展示与处理。
5. 字段失败订单、待补材料和未匹配文件均有明确入口和闭环操作，不出现“必填材料缺失但无法补传”的断点。
