# 案件导入二期设计说明

## 1. 目标与边界

二期在一期 Excel 导入能力之上补齐材料包上传、材料解析、规则配置和导入处理闭环，不改变一期已经生成并生效案件的可用状态。

二期上线后，案件导入管理与新建导入批次恢复为最终目标态；导入处理详情、导入规则配置和新建导入规则作为独立菜单承接处理与配置。

本次不引入资产方字段别名。各资产方仍使用统一的中文列头 Excel 模板；字段必填、材料类别和材料必填规则由导入规则配置维护。

## 2. 核心口径

### 2.1 案件激活状态

字段校验通过后，系统创建案件记录，并根据材料校验结果确定案件是否生效。

| 场景 | 案件状态 | 是否进入待委外 | 可操作项 |
| --- | --- | --- | --- |
| 字段校验失败、主续关系失败、金额异常、重复订单不可覆盖 | 导入失败 | 否 | 下载明细；修正 Excel 后重新导入 |
| 字段校验通过，必填材料齐全 | 已生效 | 是 | 后续法诉流程；可补充非必填材料 |
| 字段校验通过，缺少任一必填材料 | 待材料激活 | 否 | 在导入处理详情批量补传必填材料；补齐后自动生效 |
| 字段校验通过，必填材料齐全，但缺非必填材料 | 已生效 + 待补非必填 | 是 | 在导入处理详情批量补传非必填材料 |

“待材料激活”是二期新增的案件状态，不等同于“导入失败”。它表达案件基础数据已经可落库，但尚未具备进入待委外的证据材料条件。

### 2.2 一期历史案件兼容与材料补齐

一期已完成导入且已生效的案件，二期上线后继续保持“已生效”，不因当时未导入材料而回溯为待材料激活，也不影响待委外、委外和后续法诉流程。

二期为一期历史批次提供“补充历史材料”入口。运营按批次发起补充时，系统默认带入该批次的外部资产方和业务类型，并由运营从可用规则中确认适用规则；确认后保存“历史补材料规则快照”，后续材料解析和缺失清单始终按该快照处理，不受规则后续修改影响。

| 历史材料状态 | 进入条件 | 案件与流程影响 |
| --- | --- | --- |
| 待初始化（一期） | 一期历史批次尚未发起补充历史材料 | 案件保持已生效，正常流转 |
| 待补必填 | 已确认规则快照，存在缺失必填材料 | 案件保持已生效，正常流转 |
| 待补非必填 | 必填材料齐全，仍缺非必填材料 | 案件保持已生效，正常流转 |
| 材料齐全 | 当前规则快照下材料解析与必填校验均通过 | 案件保持已生效，正常流转 |

1. 一期历史批次默认显示“待初始化”，不自动生成全量待办，避免二期上线时产生大量材料任务。
2. 历史案件补充材料只能通过材料包批量上传、自动识别和未匹配材料重新识别完成；不提供人工绑定订单、重新绑定或单文件上传。
3. 补充历史材料时，必填和非必填材料均可上传；材料解析通过后，运营可在法诉材料模块沿用现有能力导出法诉材料。
4. 一期导入案件在补充材料前不支持导出法诉材料；二期需支持一期存量案件上传材料包。案件导入模块不新增导出入口、导出资格字段或导出文件。
5. 一期原有字段校验、重复订单、主续订单及金额校验口径保持不变；历史补材料不覆盖订单字段，也不改变案件生效状态。

## 3. 页面与交互调整

### 3.1 案件导入管理

恢复材料包、导入处理详情、草稿、材料状态、下载明细等目标态能力。列表新增或统一以下口径：

| 字段/动作 | 二期口径 |
| --- | --- |
| 批次状态 | 草稿、导入中、导入完成、导入失败、已取消。批次中存在成功或待材料激活订单即为“导入完成”；全部订单均未处理成功时为“导入失败”。 |
| 案件生效情况 | 已生效、待材料激活、导入失败。展示订单数量汇总，可进入导入处理详情查看订单明细。 |
| 材料状态 | 二期新批次统一为材料齐全、待补必填、待补非必填；一期历史批次初始展示待初始化（一期）。异常文件为文件维度，不混入订单或材料状态。 |
| 编辑 | 仅修改批次级信息并查看历史上传文件；订单Excel、材料包的增量提交统一从导入处理详情的“批量补传”发起。历史文件只追加，不覆盖原始上传记录。 |
| 导入处理详情 | 对导入中、导入完成和导入失败批次可进入；按订单结果、待补材料、异常文件统一处理。 |

增量导入先按批次范围判断重复：不同批次中同一“外部资产方 + 订单编号”直接阻断并提示原批次号；仅同批次重复订单继续判断流转状态。未分配且未流转的订单自动覆盖字段和同名文件，不同名文件自动补充；已分配或已流转订单不可覆盖，写入结果明细并提示当前流转状态。

### 3.2 新建导入批次

新增导入规则选择和材料包上传。资产明细和材料包均为提交导入的必传文件：任一文件未上传时整批阻断，不创建导入任务、不生成案件。保存草稿仅保存批次基础信息与临时文件，不解析、不生成案件。提交导入后系统固化规则快照，后续规则修改不影响该批次的字段和材料校验结果。

材料包上传支持 zip、rar、7z；系统按订单编号、资产方材料名称、匹配模式（精确/包含）和标准材料类别解析。解析失败、无法识别订单或无法匹配类别的文件进入未匹配材料池，不阻断其他文件的解析。待材料激活仅指材料包已上传、字段校验通过但包内缺少必填材料的订单。

### 3.3 导入处理详情

将字段和材料问题收敛在一个页面，使用三个页签；一期历史批次点击“补充历史材料”后，使用同一处理详情页面：

| 页签 | 内容 | 可操作项 |
| --- | --- | --- |
| 订单结果 | 已生效、待材料激活、导入失败、同批次覆盖等订单处理结果 | 查看解析、失败原因、批量补传、导出当前筛选订单明细 |
| 待补材料 | 待材料激活订单缺失的必填材料，以及已生效订单缺失的非必填材料 | 批量补传；补必填材料后触发材料复核与自动生效；导出当前筛选待补材料明细 |
| 异常文件 | 已上传但无法归属订单或无法归类的文件 | 预览/下载文件、批量补传后同步重新识别；导出当前筛选异常文件明细；不提供人工绑定或单文件上传 |

“批量补传”根据当前页签自动限定可上传范围：

1. 在待材料激活订单下，可补传当前规则配置的缺失必填材料和非必填材料。
2. 在已生效订单下，仅可补传非必填材料，不改变案件生效状态。
3. 失败订单的字段问题通过“批量补传”上传订单 Excel 处理；可同时上传材料包，至少上传一个文件。单次订单 Excel 最多 500 条。
4. 一期历史批次首次补充材料时，仅允许上传材料包；先确认适用规则并创建历史补材料规则快照，案件始终保持已生效。
5. 任务级异常不新增第四个页签：在详情顶部提示，并在“批次处理明细”的任务级异常工作表完整导出。

### 3.4 导入规则配置与新建规则

规则按“外部资产方 + 业务类型（租赁）”维护。新增资产方时以统一 70 列模板和标准材料类别生成初始规则，运营只编辑启用/必填及资产方材料名称，不可删除系统锁定基础字段和标准材料类别。

Excel 模板字段配置展示“模板字段 / 对应表名与字段名 / 是否启用及必填”三类信息。订单编号、姓名、身份证号、债权金额、违约日为锁定启用且必填；商品名称、下单手机号、业务主体1、主合同签署日期、租赁物签收日为默认必填，可按规则调整。

首租与续租只在材料规则配置中切换。两类订单共用同一套 Excel 字段配置，但可配置不同的必填材料清单。规则在批次提交时生成快照，后续修改仅对新批次生效。

## 4. 主续订单与重复订单

1. 首次导入续租订单时，关联主订单必须同批次导入、为已结清状态且客户证件号一致；主订单以已结清历史订单导入，不生成债权金额、已还金额、剩余未还、违约日、逾期天数等法诉债权字段。
2. 若主订单已在法诉系统形成案件，后续导入其续租订单时阻断，订单明细提示：`续租订单关联的主订单【{父订单编号}】已在法诉系统形成案件，不符合续租订单导入条件。`
3. 同一 Excel 内订单编号重复时，上传阶段阻断整份文件，不创建导入任务，提示“模板中有重复订单，请修正后重新上传。”
4. 同一批次多次补传 Excel 时，订单数量以订单编号去重统计；不可覆盖订单写入结果明细并展示原批次号和案件流转状态。不同批次重复不进入自动覆盖判断。

## 5. 备注呈现要求

二期相关页面右侧备注以 `【二期调整】` 标识新增或变化内容，至少写清：

1. 页面筛选字段和重点列表字段的取数逻辑。
2. 案件生效、待材料激活、待补必填、待补非必填、异常文件和导入失败的判定口径。
3. 每个按钮的可操作状态、处理结果与禁止条件。
4. 与一期历史批次的兼容规则，明确其可补充历史材料且案件继续保持已生效。
5. 历史材料状态和“补充历史材料”按钮的取数逻辑：首次补充时固化规则快照；材料解析通过后，可在法诉材料模块沿用现有能力导出法诉材料。
6. 规则快照、重新导入、材料补传和自动生效的边界。

## 6. 验收标准

1. 缺必填材料的订单可在处理详情补齐，补齐且复核通过后自动变为已生效并进入待委外。
2. 缺非必填材料的订单保持已生效，仅展示待补非必填，不影响后续流程。
3. 一期历史已生效案件可按批次发起补充历史材料；补充过程中案件继续已生效，不影响现有法诉流程。
4. 历史批次首次确认适用规则后保存历史补材料规则快照；规则修改后，历史材料清单仍按该快照展示与处理。
5. 一期历史案件补齐必填材料并解析通过后，可在法诉材料模块沿用现有能力导出法诉材料；案件导入模块不新增导出入口。
6. 字段失败订单、待补材料和未匹配文件均有明确入口和闭环操作，不出现“必填材料缺失但无法补传”的断点。
